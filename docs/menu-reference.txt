CAPTURED OK:
- File-type-specific extensions (txt, png, mp4, mp3, pdf, zip, exe, lnk)
- Folder-as-item menu items (Open, Open in new tab, Pin to Quick access, etc.)
- Folder background verbs (Open with Visual Studio, PowerShell, WSL, etc.)
- All built-in verbs (Cut, Copy, Paste(*), Rename, Delete, Create shortcut, ...)
- Many shell extensions via registry merge (Microsoft Defender's "EPP",
  BitLocker family, NVIDIA, Recuva, Revo, etc.)
- Items hidden by LegacyDisable (registry merge sees them)

(*) Paste captured only when the clipboard had Cut/Copy data at scan time.

STILL NOT CAPTURED — and why:

A. Explorer-internal items (FUNDAMENTALLY uncapturable)
   View, Sort by, Group by, Refresh, Customise this folder...
   These are emitted by CDefView (Explorer's IShellView), not by
   any registered IContextMenu. No registry path leads to them, no
   classic IContextMenu surfaces them. Only Explorer can produce
   them. Out of scope for RCMM.

B. Modern PackagedCom context menus  (Plan 3 territory)
   AMD Software: Adrenalin Edition
   Open in Terminal
   WinRAR
   Pin to taskbar (Start Menu Pin CLSID is packaged)
   These register under HKLM\SOFTWARE\Classes\PackagedCom\Package\<pkg>
   and are invoked by Explorer through IExplorerCommand from a sparse
   AppX manifest. Classic SHCreate / BindToHandler / GetUIObjectOf
   does NOT instantiate them. Plan 3 covers Win11 modern menu.
   Roughly 36 packages on this machine -- enumerate via:
     HKLM\SOFTWARE\Classes\PackagedCom\Package\*\Class\*
   then resolve DisplayName from the package manifest.

C. Items with non-friendly registry display
   "Microsoft Security Client Shell Extension" (=> Scan with Defender)
   "Windows Shell Common Dll" (=> various rendered by shell32.dll)
   The friendly text is the runtime QueryContextMenu emission. Without
   a live invocation of the shellex with a real target, only the DLL's
   FileDescription is available. Live capture surfaces the friendly
   name; registry-only fallback uses the technical one.

KEY BINDING STRATEGIES (confirmed working):
- shellitem-bindToHandler          IShellItem.BindToHandler(SFUIObject,IContextMenu)
- parent-getUIObjectOf             SHBindToParent + IShellFolder.GetUIObjectOf
- folder-background                SHGetDesktopFolder + BindToObject + CreateViewObject

KEY MARSHAL FIXES required during iteration:
- IShellFolder.GetUIObjectOf       IntPtr[] => ref IntPtr (AV in SafeArray marshaller)
- IContextMenu.GetCommandString    byte[]   => IntPtr  (AV in SafeArray marshaller)
- CMF_INCLUDESTATIC bogus constant  (0x40 is actually CMF_NODEFAULT - was suppressing)
- CMF_CANRENAME added              so shellexes that gate on rename support contribute

HARD CONSTRAINTS:
- IContextMenu must be queried on an STA thread with a message pump.
- 30s overall timeout protects against wedged shellexes.
- Sample files must be non-empty (>=1 byte) or some shellexes skip them.
